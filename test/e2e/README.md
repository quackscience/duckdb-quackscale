# Headscale QuackTail e2e

Integration tests for a two-node QuackTail cluster over [Headscale](https://github.com/juanfont/headscale).

## Canonical path (matches examples demo)

```bash
git submodule update --init --recursive
chmod +x scripts/ci_compose_e2e.sh
./scripts/ci_compose_e2e.sh
```

Same steps as [examples/README.md](../../examples/README.md):

1. `docker compose build` (source, `BUILD_FROM_SOURCE=1`)
2. `quacktail-verify-image.sh` (`attach_ducklake`, `tailscale_down`, `tailscale_quack_forward`)
3. `docker compose up -d headscale quacktail-server`
4. `docker compose --profile test run --rm quacktail-client`

Expect `LAKE_PASSED`, `PASSED`, `attach_ducklake`, and `✓ Demo passed`.

## Legacy host-container e2e

[`scripts/ci_headscale_e2e.sh`](../../scripts/ci_headscale_e2e.sh) runs concurrent server/client containers with a bind-mounted host `duckdb` binary and [`test/e2e/Dockerfile.quacktail`](Dockerfile.quacktail). Prefer **compose e2e** above — it uses [`examples/Dockerfile`](../../examples/Dockerfile) and the same entrypoint/bootstrap as the documented demo.

## Client session shape

Generated by [`scripts/e2e/quacktail-compose-bootstrap.sh`](../../scripts/e2e/quacktail-compose-bootstrap.sh):

```sql
LOAD quackscale;
CALL tailscale_up(...);
CALL tailscale_quack_forward(host => 'quacktail-server', port => 9494, local_port => 19494);
CALL tailscale_ping(...);
LOAD quack; CREATE SECRET ...;
FROM quack_query(..., 'SELECT 1 AS probe', ...);
CALL attach_ducklake(...);
SELECT * FROM lake.inventory ...;
SELECT 'LAKE_PASSED' ...;
ATTACH 'quack:127.0.0.1:19494' AS remote (TYPE quack);
SELECT 'PASSED' ...;
SELECT 'CLIENT_DEMO_DONE' AS status;
CALL tailscale_down();
```

## Server (`loopback_serve`)

```sql
CALL quack_serve('quack:127.0.0.1:9494', allow_other_hostname => true, token => quack_token());
CALL tailscale_serve_local(port => 9494);
```

Healthcheck: `/work/server.log` contains `quack:127.0.0.1:9494` and `local_forward`.
